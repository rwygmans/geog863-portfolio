Reflection

I built a simple hurricane viewer that asks for a year and draws IBTrACS track lines for that period. The map uses the dark-gray-vector basemap and a single FeatureLayer for the tracks. I filter the layer with a definitionExpression on the year field so only that year’s features appear. Tracks are shown as dotted lines whose width and color change with Saffir–Simpson categories via a ClassBreaksRenderer on USA_WIND. I went with a yellow‑orange‑red palette and progressively thicker strokes so the strongest segments stand out without overwhelming the basemap.

Popups include storm name, year, maximum wind speed, basin, nature, and date using a fields‑based PopupTemplate. After the layer loads, I run a quick Query with the same year filter to confirm that data actually exists. If nothing comes back (or the request fails), the app nudges the user rather than leaving an empty map. The flow is pretty straightforward: import modules with $arcgis.import, make the map and view, define the renderer, add the layer with its popup and filter, and finally validate the results.

The trickiest parts were picking class breaks and stroke widths that read well at different zoom levels and user input error handling. Validating the year and using numeric comparisons in the definitionExpression helped avoid odd cases. I also tweaked the dotted SimpleLineSymbol so categories feel distinct but not noisy. I came away more comfortable with ClassBreaksRenderer, FeatureLayer filtering, Query patterns, and PopupTemplate fieldInfos. My main references were the Esri docs, API sandbox samples, and ColorBrewer for symbology.
